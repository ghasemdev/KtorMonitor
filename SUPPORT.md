### Asking a Good Question

When asking for help, please include:

1. **KtorMonitor version** (and the relevant artifact: `ktor-monitor-logging`, `ktor-monitor-okhttp-interceptor`, etc.).
2. **Platform / target** (Android, iOS, JVM Desktop, JS, Wasm).
3. **Kotlin version** and **Compose Multiplatform version**.
4. A **minimal reproducible example** — the smallest code that demonstrates the issue.
5. **Expected behaviour** vs **actual behaviour**.
6. Relevant **logs or stack traces** (redact sensitive headers / credentials).

## Reporting a Bug

1. Search [existing issues](https://github.com/CosminMihuMDC/KtorMonitor/issues) first — the bug may already be reported or fixed.
2. If not found, [open a new issue](https://github.com/CosminMihuMDC/KtorMonitor/issues/new) and fill in the bug report template.
3. Include the information listed in **Asking a Good Question** above.

> **Security vulnerabilities** must **not** be reported via public issues.  
> Please follow the process described in [`SECURITY.md`](SECURITY.md).

---

## Requesting a Feature

1. Search [existing issues](https://github.com/CosminMihuMDC/KtorMonitor/issues) and [discussions](https://github.com/CosminMihuMDC/KtorMonitor/discussions) to avoid duplicates.
2. [Open a new issue](https://github.com/CosminMihuMDC/KtorMonitor/issues/new) using the feature request template.
3. Describe the **use case**, not just the solution — explain the problem you are trying to solve.

## Response Time

This is an open-source project maintained on a best-effort basis.  
There is no guaranteed response time, but maintainers aim to triage new issues within a reasonable timeframe.  
Community members are also encouraged to help answer questions on Slack.
